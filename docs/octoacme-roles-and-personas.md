# OctoAcme Personas

This document defines typical roles and responsibilities used in OctoAcme project docs and exercises.

---

## Developers

### Role Summary
Developers design, build, test, and deliver software components. They collaborate with product and project leads to implement features that meet acceptance criteria and quality standards.

### Responsibilities
- Implement features and fixes to meet acceptance criteria
- Write and maintain tests and documentation
- Participate in design and code reviews
- Assist in estimating and planning work
- Help identify technical risks and propose mitigations

### Goals
- Deliver reliable, maintainable code
- Reduce cycle time from idea to production
- Maintain high test coverage and observability

### Typical Communication
- Daily standups and sprint planning
- PR descriptions and code review comments
- Technical design docs when needed

---

## Product Managers

### Role Summary
Product Managers define what should be built to deliver customer and business value. They own the product vision, prioritize the backlog, and measure outcomes.

### Responsibilities
- Define problem statements and success metrics
- Prioritize the roadmap and backlog
- Collaborate with stakeholders and engineering on trade-offs
- Validate solutions through user research and metrics

### Goals
- Maximize customer value and impact
- Make clear, data-driven prioritization decisions
- Ensure product-market fit and usability

### Typical Communication
- Weekly alignment with PM and engineering leads
- Roadmap updates and stakeholder briefings
- Acceptance criteria and feature specs

---

## Project Managers

### Role Summary
Project Managers coordinate delivery activities, manage schedules, risks, and communications. They enable the team to deliver on commitments efficiently.

### Responsibilities
- Create and maintain project plans and timelines
- Manage risks, dependencies, and resource constraints
- Facilitate meetings (kickoff, planning, retrospectives)
- Ensure consistent project documentation and status reporting
- Coordinate cross-team and stakeholder communication

### Goals
- Deliver projects on time and within scope
- Minimize unplanned work and escalations
- Maintain transparency and alignment across stakeholders

### Typical Communication
- Weekly status updates and stakeholder reports
- Risk registers and decision logs
- Coordination via project boards and meeting facilitation

---

## UX/UI Designer

### Role Summary
UX/UI Designers create user-centered designs that balance usability, accessibility, and visual appeal. They conduct user research, create wireframes and prototypes, and collaborate with Product Managers and Developers to deliver intuitive user experiences.

### Responsibilities
- Conduct user research and usability testing to inform design decisions
- Create wireframes, mockups, and interactive prototypes
- Design consistent and accessible user interfaces following design systems
- Collaborate with Product Managers to understand user needs and business goals
- Work with Developers to ensure designs are implemented correctly
- Maintain design documentation and component libraries
- Advocate for user experience throughout the product lifecycle

### Goals
- Deliver intuitive, accessible designs that meet user needs
- Reduce user friction and improve satisfaction metrics
- Maintain design consistency across the product
- Balance user needs with technical constraints and business objectives

### Typical Communication
- Participate in product planning and feature kickoffs
- Present design concepts and research findings to stakeholders
- Collaborate daily with Developers during implementation
- Share design updates in weekly syncs
- Conduct design critiques and feedback sessions with the team

### Interactions with Other Roles
- **Product Managers:** Collaborate on user needs, feature prioritization, and acceptance criteria
- **Developers:** Pair on implementation details and technical feasibility
- **QA/Testing:** Define visual and interaction acceptance criteria
- **Technical Writers:** Ensure UI language aligns with documentation
- **Customer Support Liaison:** Gather user feedback and pain points

---

## Technical Writer/Documentation Specialist

### Role Summary
Technical Writers create and maintain clear, accurate, and user-friendly documentation for internal teams, end users, and external stakeholders. They ensure documentation stays up-to-date and accessible throughout the product lifecycle.

### Responsibilities
- Write and maintain user guides, API documentation, and release notes
- Create internal process documentation and runbooks
- Collaborate with Developers to document technical features accurately
- Ensure documentation follows style guides and accessibility standards
- Organize and structure documentation for easy discovery
- Review documentation during code reviews and releases
- Gather feedback to continuously improve documentation quality

### Goals
- Provide accurate, accessible documentation that reduces support burden
- Keep documentation synchronized with product changes
- Enable self-service for users and team members
- Maintain consistent voice and style across all documentation

### Typical Communication
- Attend planning meetings to understand upcoming features
- Review PRs and technical designs for documentation needs
- Coordinate with Product Managers on release communications
- Collaborate with Customer Support on common user questions
- Share documentation updates in release notes and team syncs

### Interactions with Other Roles
- **Developers:** Review technical accuracy and gather implementation details
- **Product Managers:** Align documentation with product vision and user needs
- **UX/UI Designers:** Ensure consistency between UI copy and documentation
- **Customer Support Liaison:** Identify documentation gaps from user feedback
- **DevOps/Platform Engineer:** Document deployment processes and infrastructure

---

## DevOps/Platform Engineer

### Role Summary
DevOps/Platform Engineers build and maintain the infrastructure, tooling, and automation that enable reliable, efficient software delivery. They focus on CI/CD pipelines, monitoring, scalability, and operational excellence.

### Responsibilities
- Design, build, and maintain CI/CD pipelines and deployment automation
- Manage infrastructure as code and cloud resources
- Implement monitoring, logging, and alerting systems
- Ensure system reliability, scalability, and performance
- Automate operational tasks and improve developer productivity
- Respond to incidents and participate in on-call rotations
- Document infrastructure architecture and operational procedures

### Goals
- Enable fast, safe, and reliable deployments to production
- Minimize downtime and improve system reliability
- Automate repetitive tasks to improve team efficiency
- Provide visibility into system health and performance

### Typical Communication
- Participate in sprint planning to estimate infrastructure work
- Provide input on technical architecture and scalability
- Share system status and incident updates with stakeholders
- Collaborate with Developers on deployment requirements
- Document infrastructure changes and operational procedures

### Interactions with Other Roles
- **Developers:** Enable efficient development workflows and deployments
- **Security Lead:** Implement security controls and vulnerability scanning
- **Project Manager:** Report on infrastructure risks and capacity planning
- **Technical Writer:** Collaborate on runbooks and operational documentation
- **QA/Testing:** Set up test environments and deployment pipelines

---

## Customer Support Liaison

### Role Summary
Customer Support Liaisons bridge the gap between users and the product team, representing customer voice in planning and ensuring user feedback informs product decisions. They triage issues, coordinate support responses, and identify improvement opportunities.

### Responsibilities
- Triage and escalate customer issues to appropriate teams
- Represent customer perspective in planning and retrospectives
- Track and analyze support trends to identify product improvements
- Coordinate with Product Managers on feature requests and bug prioritization
- Document known issues and workarounds for users and support teams
- Provide input on release communications and customer impact
- Collaborate on customer-facing documentation and FAQs

### Goals
- Reduce time-to-resolution for customer issues
- Improve product quality by surfacing user feedback
- Enhance customer satisfaction and retention
- Build strong relationships between support and product teams

### Typical Communication
- Daily triage of support tickets and escalations
- Weekly sync with Product Managers on trends and priorities
- Participate in release planning for customer impact assessment
- Share customer feedback in retrospectives
- Coordinate with Technical Writers on support documentation

### Interactions with Other Roles
- **Product Managers:** Share customer insights and feature requests
- **Developers:** Escalate bugs and provide reproduction steps
- **QA/Testing:** Coordinate on testing customer-reported issues
- **Technical Writer:** Identify documentation gaps from support inquiries
- **Project Manager:** Provide input on customer impact and release timing

---

## Security Lead

### Role Summary
Security Leads ensure security is integrated throughout the software development lifecycle, from design through deployment. They assess risks, define security requirements, and guide the team in building secure, compliant systems.

### Responsibilities
- Define security requirements and acceptance criteria for features
- Conduct threat modeling and security reviews for designs and implementations
- Oversee security scanning tools and vulnerability management
- Respond to security incidents and coordinate remediation
- Ensure compliance with security policies and regulations
- Educate team members on secure coding practices
- Maintain security documentation and incident response procedures

### Goals
- Minimize security vulnerabilities and reduce attack surface
- Ensure timely remediation of identified security issues
- Build security awareness and capability across the team
- Maintain compliance with regulatory and organizational requirements

### Typical Communication
- Review security aspects during planning and design reviews
- Participate in code reviews for security-sensitive changes
- Provide security updates in weekly syncs and status reports
- Conduct security training and awareness sessions
- Coordinate incident response with DevOps and management

### Interactions with Other Roles
- **Developers:** Guide secure coding practices and review implementations
- **DevOps/Platform Engineer:** Define security controls for infrastructure and CI/CD
- **Product Managers:** Balance security requirements with feature priorities
- **Project Manager:** Report on security risks and remediation timelines
- **QA/Testing:** Define security test cases and penetration testing scope
