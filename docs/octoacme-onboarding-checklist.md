## Purpose
This checklist helps new team members and existing team members joining a new project get up to speed quickly with clear ownership, communication expectations, and access to necessary resources.

## For New Team Members

### First Day
- [ ] Complete company onboarding and access setup
- [ ] Receive invitation to project repositories and communication channels
- [ ] Review [Project Management Overview](octoacme-project-management-overview.md)
- [ ] Understand your role using [Roles & Personas](octoacme-roles-and-personas.md)
- [ ] Meet your buddy/mentor assigned by Project Manager
- [ ] Schedule 1:1 introductions with key team members

### First Week
- [ ] Read project documentation in the `docs/` directory
- [ ] Review current Project One-pager and success metrics
- [ ] Attend daily standups and team syncs
- [ ] Set up development environment (if applicable)
- [ ] Review project board and understand workflow states
- [ ] Understand communication cadence and meeting schedule
- [ ] Review risk register and current project status

### First Month
- [ ] Complete first meaningful contribution (code, design, documentation, etc.)
- [ ] Participate in sprint planning or project planning session
- [ ] Attend and contribute to a retrospective
- [ ] Understand escalation paths and incident response procedures
- [ ] Shadow experienced team members in your role
- [ ] Provide feedback on onboarding experience

## Role-Specific Onboarding

### Developers
- [ ] Set up local development environment and IDE
- [ ] Review code style guide and PR conventions
- [ ] Understand CI/CD pipeline and testing requirements
- [ ] Review architecture documentation and technical design docs
- [ ] Complete first code review (as reviewer and reviewee)
- [ ] Run tests locally and understand test coverage expectations

### UX/UI Designers
- [ ] Access design tools and component libraries
- [ ] Review design system and accessibility guidelines
- [ ] Understand user research processes and feedback channels
- [ ] Review existing designs and prototypes
- [ ] Shadow user research session or usability test
- [ ] Collaborate on a design review session

### Technical Writers
- [ ] Access documentation tools and repositories
- [ ] Review style guide and documentation standards
- [ ] Understand documentation review process
- [ ] Identify documentation gaps and improvement opportunities
- [ ] Coordinate with Developers on upcoming features
- [ ] Review release notes process and templates

### DevOps/Platform Engineers
- [ ] Access to infrastructure and cloud platforms
- [ ] Review infrastructure as code repositories
- [ ] Understand monitoring, logging, and alerting setup
- [ ] Review incident response procedures and on-call schedule
- [ ] Understand deployment processes and rollback procedures
- [ ] Review security and compliance requirements

### Customer Support Liaisons
- [ ] Access to support ticketing system and knowledge base
- [ ] Review escalation procedures and SLA expectations
- [ ] Understand product features and common user issues
- [ ] Shadow experienced support team members
- [ ] Review customer communication templates
- [ ] Join customer feedback and product planning sessions

### Security Leads
- [ ] Review security policies and compliance requirements
- [ ] Access security scanning tools and vulnerability databases
- [ ] Understand threat modeling and security review processes
- [ ] Review incident response and security escalation procedures
- [ ] Conduct security assessment of current systems
- [ ] Schedule security training for development team

### QA/Testing
- [ ] Access test environments and testing tools
- [ ] Review test strategy and coverage requirements
- [ ] Understand acceptance criteria and Definition of Done
- [ ] Review bug tracking and reporting procedures
- [ ] Understand release testing and smoke test procedures
- [ ] Collaborate on test planning for upcoming features

### Product Managers
- [ ] Review product vision, roadmap, and backlog
- [ ] Understand success metrics and analytics tools
- [ ] Meet with key stakeholders and customers
- [ ] Review market research and competitive analysis
- [ ] Understand prioritization framework
- [ ] Schedule regular syncs with Project Manager and team leads

### Project Managers
- [ ] Review project charter and current project status
- [ ] Access project boards, tools, and tracking systems
- [ ] Understand resource allocation and team capacity
- [ ] Review risk register and active issues
- [ ] Meet all team members and stakeholders
- [ ] Schedule regular syncs and establish communication cadence

## For Existing Team Members Joining a Project

### Week Before Project Start
- [ ] Review Project One-pager and charter
- [ ] Understand project goals and success metrics
- [ ] Review proposed timeline and milestones
- [ ] Identify dependencies on your work or expertise
- [ ] Block calendar for project meetings and syncs

### Project Kickoff
- [ ] Attend project kickoff meeting
- [ ] Understand your specific responsibilities and deliverables
- [ ] Review initial backlog and acceptance criteria
- [ ] Identify knowledge gaps and schedule knowledge transfer sessions
- [ ] Commit to project communication cadence
- [ ] Add project tasks to your personal work tracking

## Onboarding Ownership

### Project Manager Responsibilities
- Assign onboarding buddy/mentor
- Schedule introductory meetings with team members
- Provide access to project documentation and tools
- Review onboarding checklist progress weekly
- Collect feedback on onboarding experience

### Team Lead Responsibilities (Role-specific)
- Provide role-specific training and shadowing opportunities
- Review role responsibilities and expectations
- Assign first meaningful task appropriate to skill level
- Schedule regular check-ins during first month
- Share best practices and team norms

### Buddy/Mentor Responsibilities
- Be available for questions and guidance
- Introduce new team member to informal team dynamics
- Help navigate tools, processes, and documentation
- Provide feedback and encouragement
- Escalate concerns to Project Manager if needed

## Continuous Improvement

After completing onboarding:
- [ ] Provide feedback on onboarding experience to Project Manager
- [ ] Suggest improvements to documentation or processes
- [ ] Update onboarding checklist based on your experience
- [ ] Volunteer to be a buddy/mentor for future team members
